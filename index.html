AI Capability Assessment

          
Embedded into the SE interview process — no extra rounds. We assess how candidates use AI as a collaborator, not just whether they can solve problems.

        

        
Principles

        

          
🪨AI complements, not replaces, engineering fundamentals

          
✅AI is a MUST — openness, application and tool use are all assessed

          
⚖️Strong AI cannot compensate for weak technical fundamentals

          
🚫Probe past surface-level answers — look for concrete examples

        

        
Hybrid Format

        

          
Traditional coding/design + AI-assisted interview. Candidates show strong technical skills and effective AI tool use.

          

            
1
Preferred — Candidate uses own AI tool + shares screen

            
2
Fallback — Interviewer shares screen to run prompts together

            
3
Last resort — Candidate types prompt verbally

          

          
TA to brief candidates in advance — ask them to have their preferred AI tool ready.

        

        
What We Evaluate

        

          
AI Understanding
LLM capabilities & limits · Hallucination/bias risks · Prompt design · When to use AI

          
AI Empowerment
AI tools in dev workflow · Validating AI outputs · Measurable productivity gains

          
AI LeadershipTL/P7+ only
Driving team adoption · Coaching on AI literacy · Setting workflow standards

Candidate Rating Guide

        

          

            
Strong Yes

            
Extensive hands-on experience with Claude Code, Cursor, or equivalent AI tools. Clear prompt iteration process, actively follows AI trends, demonstrates team promotion mindset.

          

          

            
Yes

            
Uses AI tools regularly, solid hands-on experience, basic awareness of current trends

          

          

            
No (Borderline)

            
Occasional AI usage, lacks systematic thinking, limited awareness of emerging trends

          

          

            
Strong No

            
Rarely or never uses AI tools, shows no motivation for self-directed learning

          

        

        

          

            
Red Flags

            

              
🚩 Uses AI buzzwords but can't explain the concept

              
🚩 Blindly accepts AI output without verification

              
🚩 No concrete examples — only hypothetical use

              
🚩 Can't explain fundamentals of AI-generated code they wrote

              
🚩 Hype-driven — AI solves everything, no acknowledgment of limits

            

          

          

            
Green Signals

            

              
✅ Specific tool + specific task + measurable outcome

              
✅ Shows prompt iteration — refined when first output wasn't good enough

              
✅ Knows when NOT to use AI — data sensitivity, legal constraints

              
✅ Proactively shares AI workflow without being prompted

              
✅ Can differentiate Claude Code, Copilot, Cursor — knows when to use which

            

          

        

      


      
      

        

          
🚀 Implementation Guide

          
To be reviewed after first 2-3 weeks post-launch.

        

        
👔 Hiring Manager

        

          

            
⏱️Interview Time Constraints
▶

            
AI capability assessment should add no more than 10-15 minutes to existing rounds. Prioritise live demonstration over verbal description. Dimension 2 (hands-on) is the highest signal — allocate time accordingly. If time is short, focus on D2 only.

          

          

            
🎓Interviewer Readiness
▶

            
Interviewers should themselves be active users of Claude Code or Cursor to credibly evaluate candidates. If you have not used these tools in the past 2 weeks, pair with someone who has. Calibration sessions recommended before first use of this framework.

          

          

            
🔧Tool Access Issues
▶

            
If candidate can't access their tool during interview, fall back to screen-share or verbal walkthrough. Do not penalise candidates for access issues. If a candidate has never used these specific tools, assess transferable prompt engineering skills with whatever tool they use daily.

          

        

        
😺 Recruiting Team

        

          

            
😺Candidate Readiness
▶

            
Brief all candidates in advance: "Part of your interview will involve using AI tools as a collaborator. Please have your preferred tool (Claude Code, Cursor, GitHub Copilot, etc.) ready and accessible. We will assess how you work with AI, not just whether you use it."

          

          

            
📋Greenhouse Scorecard Setup
▶

            
Add "AI Capability" as a scored attribute in Greenhouse scorecards for all Engineering roles. Use the 4-point rating (Strong Yes / Yes / No (Borderline) / Strong No). Tag the interviewer role (Coding, System Design, VP) so scores are weighted correctly in aggregation. Contact TA Ops to configure.
